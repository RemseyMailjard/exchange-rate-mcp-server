1. Install uv on Windows or macOS

On Windows, open PowerShell and run:

PowerShell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"

Restart your terminal and verify the installation:

PowerShell
uv --version

Optional: install Python through uv:

PowerShell
uv python install 3.12

On macOS, you can install uv with Homebrew or the official install script.

Homebrew:

Terminal
brew install uv

Official install script:

Terminal
curl -LsSf https://astral.sh/uv/install.sh | sh

Restart your terminal and verify the installation:

Terminal
uv --version

Optional: install Python through uv:

Terminal
uv python install 3.12

2. Create a new MCP project

Navigate to the folder where you want to create the project. Choose any location that works for you, for example your Documents or Desktop folder.

Windows · PowerShell
cd "C:\Users\YourName\Documents"
macOS / Linux · Terminal
cd ~/Documents

Create a new project:

PowerShell
uv init first-mcp-server
cd first-mcp-server

This creates a Python project with a pyproject.toml file.

3. Add FastMCP

Install FastMCP into the project:

PowerShell
uv add fastmcp

For this minimal demo, fastapi and requests are not needed.

4. Create the minimal MCP server

Open main.py and start with this minimal server:

Python
from fastmcp import FastMCP

mcp = FastMCP("Rabobank Demo MCP Server")


@mcp.tool()
def get_account_balance(account_number: str) -> str:
    """Get the balance for an internal Rabobank account."""
    return f"Account {account_number} has a balance of €1,250.00"


if __name__ == "__main__":
    mcp.run()

This server exposes one MCP tool:

Text
get_account_balance

5. Run the MCP server over HTTP

Run the server with HTTP transport:

PowerShell
uv run fastmcp run main.py:mcp --transport http --port 8000

You should see output similar to:

Terminal output
Starting MCP server 'Rabobank Demo MCP Server'
with transport 'http' on http://127.0.0.1:8000/mcp
Uvicorn running on http://127.0.0.1:8000
Server is running When you see Uvicorn running on http://127.0.0.1:8000 the MCP server is active. Leave this terminal window open — closing it stops the server.
Terminal shows nothing or exits immediately? Check that you are inside the first-mcp-server folder and that uv add fastmcp completed successfully.

6. Important: do not test /mcp as a normal webpage

If you open this URL in the browser:

URL
http://127.0.0.1:8000/mcp

You may see:

JSON response
{
  "jsonrpc": "2.0",
  "id": "server-error",
  "error": {
    "code": -32600,
    "message": "Not Acceptable: Client must accept text/event-stream"
  }
}
This is expected. The /mcp endpoint is not a normal website or REST API endpoint. It is meant to be used by an MCP-compatible client.

7. Connect the MCP server to VS Code

Before you continue: open the project folder in VS Code VS Code must have the first-mcp-server folder open as a workspace. Use File → Open Folder and select first-mcp-server. Opening just one file is not enough — the .vscode/ folder must be at the workspace root.

In your project, create this file:

File path
.vscode/mcp.json

Add the following configuration:

JSON
{
  "servers": {
    "rabobank-demo": {
      "url": "http://127.0.0.1:8000/mcp"
    }
  }
}

Save the file.

Your HTTP server must already be running at this point The MCP server from step 5 should still be active in a separate terminal. VS Code connects to it over HTTP — if the server is stopped, VS Code cannot reach it.

8. Start the MCP server in VS Code

Follow these steps in order:

  1. 1
    Open the project folder In VS Code, use File → Open Folder and select first-mcp-server. This is required — VS Code needs the folder open so it can detect .vscode/mcp.json.
  2. 2
    Open .vscode/mcp.json In the Explorer panel, open the file. VS Code shows a Start button (CodeLens) directly above the server entry in the file.
  3. 3
    Click Start VS Code connects to the running HTTP server on port 8000 and registers the MCP tools.
  4. 4
    Verify the connection Open GitHub Copilot Chat, switch to Agent mode, and click the Select tools icon (⊕). You should see rabobank-demo listed with its available tools.
MCP server is connected When rabobank-demo appears in the tools list in GitHub Copilot Chat, the connection is active. GitHub Copilot can now discover and call the MCP tools.
Start button not visible in mcp.json? Make sure you opened the folder in VS Code, not just the file. Also check that the GitHub Copilot extension is installed and that you are signed in.
Tools not visible in Copilot Chat? Confirm that the terminal from step 5 still shows Uvicorn running on http://127.0.0.1:8000. If not, restart the MCP server and click Start again in mcp.json.

Goal

Understand how an MCP server works and how AI assistants can securely interact with internal business systems through MCP tools.

Imagine you are a developer at a bank. Many internal systems contain useful information, but AI assistants cannot access them directly.

An MCP server acts as a controlled bridge between the AI assistant and internal systems. In this lab, you create your first MCP server and expose a few internal banking operations as MCP tools.

By the end of this exercise, GitHub Copilot should be able to discover and call your tools through the MCP protocol.

Task 1. Create the Account Balance Tool

Implement a tool that returns a balance for a given account number.

Python
ACCOUNTS = {
    "12345": {"customer_id": "1001", "balance": 1250.00, "currency": "EUR"},
    "67890": {"customer_id": "1002", "balance": 2480.75, "currency": "EUR"},
}


@mcp.tool()
def get_account_balance(account_number: str) -> str:
    account = ACCOUNTS.get(account_number)

    if account is None:
        return f"No account found for {account_number}."

    return (
        f"Account {account_number} has a balance of "
        f"{account['currency']} {account['balance']:.2f}"
    )
Example
Account 12345 has a balance of EUR 1250.00

Test the tool locally after adding it.

Task 3. Create the Branch Information Tool

Implement a tool that returns branch information.

Python
BRANCHES = {
    "BR001": {
        "location": "Utrecht",
        "opening_hours": "09:00 - 17:00",
        "services": ["Daily banking", "Mortgage advice", "Business support"],
    }
}


@mcp.tool()
def get_branch_information(branch_code: str) -> str:
    branch = BRANCHES.get(branch_code)

    if branch is None:
        return f"No branch found for {branch_code}."

    services = ", ".join(branch["services"])
    return (
        f"Branch {branch_code}\n"
        f"Location: {branch['location']}\n"
        f"Opening Hours: {branch['opening_hours']}\n"
        f"Services: {services}"
    )
Example
Branch BR001
Location: Utrecht
Opening Hours: 09:00 - 17:00
Services: Daily banking, Mortgage advice, Business support

Challenge Exercise

Add a new tool that returns a demo exchange rate in EUR:

Python
EXCHANGE_RATES = {
    "USD": "1 USD = 0.92 EUR",
    "GBP": "1 GBP = 1.17 EUR",
    "CHF": "1 CHF = 1.04 EUR",
}


@mcp.tool()
def get_exchange_rate(currency: str) -> str:
    normalized_currency = currency.upper()
    exchange_rate = EXCHANGE_RATES.get(normalized_currency)

    if exchange_rate is None:
        supported = ", ".join(sorted(EXCHANGE_RATES))
        return (
            f"No exchange rate found for {normalized_currency}. "
            f"Supported currencies: {supported}."
        )

    return exchange_rate

Examples:

GBP -> EUR
CHF -> EUR

Bonus Exercise: Add a Resource to the MCP service

Now that you have a working MCP server, add a human-readable resource file and expose it so the AI can retrieve it. Use the example in resources/exchange-rate-resource.md.

Steps

  1. Place the resource file in the resources/ folder (the repo already includes resources/exchange-rate-resource.md).
  2. Add a small MCP tool that returns the contents of a resource file. Example for main.py:
Python
@mcp.tool()
def get_resource(resource_name: str) -> str:
    """Return the contents of a Markdown resource from the resources/ folder."""
    safe_name = resource_name.replace('..', '').replace('/', '')
    path = f"resources/{safe_name}.md"
    try:
        with open(path, encoding='utf-8') as f:
            return f.read()
    except FileNotFoundError:
        return f"Resource '{resource_name}' not found."

Example usage in Copilot Chat:

Prompt
Use the rabobank-demo MCP server to show the exchange-rate-resource.

This approach keeps resources as plain files that your MCP server can safely serve to approved AI clients. For production, validate and sanitize resource names and consider a permissions layer.

Bonus Exercise: Live API Integration

This bonus shows how to turn a live third-party API into an MCP tool. The repo includes a demo tool get_public_exchange_rate in main.py that calls exchangerate.host.

What to add

  1. Review the get_public_exchange_rate(from_currency, to_currency) tool in main.py.
  2. Start the MCP server and test the tool using Copilot Chat (Agent mode) or by calling the tool directly from an MCP client.
Python (example tool)
@mcp.tool()
def get_public_exchange_rate(from_currency: str, to_currency: str = "EUR") -> str:
    """Fetch live exchange rate from exchangerate.host and return a readable result."""
    from_currency = from_currency.upper()
    to_currency = to_currency.upper()
    url = f"https://api.exchangerate.host/convert?from={from_currency}&to={to_currency}"
    resp = httpx.get(url, timeout=10.0)
    resp.raise_for_status()
    data = resp.json()
    rate = data.get("info", {}).get("rate")
    if rate is None:
        return "Unexpected response from public exchange rate API."
    return f"1 {from_currency} = {rate} {to_currency} (source: exchangerate.host)"

Example Copilot prompt:

Prompt
Use the rabobank-demo MCP server to get the live USD to EUR rate using the public API tool.

Tip: When integrating live APIs, handle network errors, timeouts and unexpected responses. For production, add caching, rate-limit handling, and API keys where required.
