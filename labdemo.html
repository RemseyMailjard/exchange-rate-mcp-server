2. Create a new MCP project

Navigate to the folder where you want to create the project. Choose any location that works for you, for example your Documents or Desktop folder.

Windows · PowerShell

macOS / Linux · Terminal
cd ~/Documents

PowerShell

cd first-mcp-server

This creates a Python project with a pyproject.toml file.

3. Add the dependencies

Install FastMCP into the project:

PowerShell
uv add fastmcp

The first minimal tool only needs FastMCP. Later in this lab you add live exchange-rate tools that call an external API and read a key from a .env file, so install those two libraries now as well:

PowerShell
uv add httpx python-dotenv

This matches the final pyproject.toml, which depends on fastmcp, httpx, and python-dotenv. You do not need fastapi or requests.

4. Create the minimal MCP server

Open main.py and start with this minimal server:

Python
from fastmcp import FastMCP

mcp = FastMCP("Rabobank Demo MCP Server")


@mcp.tool()
def get_account_balance(account_number: str) -> str:
    """Get the balance for an internal Rabobank account."""
    return f"Account {account_number} has a balance of €1,250.00"


if __name__ == "__main__":
    mcp.run()

Best practice: give every tool a docstring

Before you build the tools, it helps to understand why each one in this lab has a docstring. A docstring is not just a comment for developers. With FastMCP, the docstring becomes the tool description that is sent to the AI client over the MCP protocol.

When GitHub Copilot decides which tool to call, it reads three things from your function:

Tool name The function name, for example get_account_balance.
Description The docstring, which explains what the tool does and when to use it.
Input schema The type hints, for example account_number: str.
Why it matters Clear names, docstrings, and types help the AI pick the right tool with the right arguments.

Compare a tool without a docstring to one with a clear docstring:

Avoid
@mcp.tool()
def get_account_balance(account_number: str) -> str:
    account = ACCOUNTS.get(account_number)
    ...
Prefer
@mcp.tool()
def get_account_balance(account_number: str) -> str:
    """Get the balance of an internal account by account number."""
    account = ACCOUNTS.get(account_number)
    ...
Tip: write the docstring from the AI's point of view. Describe what the tool returns and when it should be used, not how it is implemented. For tools with several arguments, document each argument in the docstring as well (see the convert_currency tool in main.py).

Task 1. Create the Account Balance Tool

Implement a tool that returns a balance for a given account number.

Python
ACCOUNTS = {
    "12345": {"customer_id": "1001", "balance": 1250.00, "currency": "EUR"},
    "67890": {"customer_id": "1002", "balance": 2480.75, "currency": "EUR"},
}


@mcp.tool()
def get_account_balance(account_number: str) -> str:
    """Get the balance of an internal account by account number."""
    account = ACCOUNTS.get(account_number)

    if account is None:
        return f"No account found for {account_number}."

    return (
        f"Account {account_number} has a balance of "
        f"{account['currency']} {account['balance']:.2f}"
    )
Example
Account 12345 has a balance of EUR 1250.00

BRANCHES = {

@mcp.tool()
def get_branch_information(branch_code: str) -> str:

Challenge Exercise: Add live exchange rate tools

The final server in main.py does not use hardcoded rates. It calls the live ExchangeRate-API so the AI can answer real currency questions. Follow these steps to reach that end result.

Step 1. Get a free API key

  1. Sign up at app.exchangerate-api.com.
  2. Confirm your email and open the dashboard.
  3. Copy your API key (it looks like 1234567890abcdef12345678).

Step 2. Store the key in a .env file

Create a file named .env in the project root:

.env
EXCHANGE_RATE_API_KEY=your_real_key_here

Make sure .env is listed in .gitignore so your key is never committed.

Step 3. Load the key and add the imports

Update the top of main.py so it loads the .env file and prepares the API call:

Python (top of main.py)
import os

import httpx
from dotenv import load_dotenv
from fastmcp import FastMCP

load_dotenv()

mcp = FastMCP("Rabobank Demo MCP Server")

EXCHANGE_RATE_API_KEY = os.getenv("EXCHANGE_RATE_API_KEY")
EXCHANGE_RATE_API_BASE = "https://v6.exchangerate-api.com/v6"
You installed httpx and python-dotenv in step 3 of Part 1. If you skipped that, run uv add httpx python-dotenv now.

Step 4. Add a shared helper that fetches live rates

All three currency tools reuse one helper. Centralizing the network call keeps the integration reliable.

Python
def _fetch_conversion_rates(base_currency: str) -> tuple[dict[str, float] | None, str | None]:
    """Fetch live conversion rates for a base currency from ExchangeRate-API.

    Returns a tuple of (rates, error_message). Exactly one element is non-None.
    """
    if not EXCHANGE_RATE_API_KEY:
        return None, (
            "API key is missing. Set EXCHANGE_RATE_API_KEY as an environment variable."
        )

    url = f"{EXCHANGE_RATE_API_BASE}/{EXCHANGE_RATE_API_KEY}/latest/{base_currency}"

    try:
        response = httpx.get(url, timeout=10.0)
        response.raise_for_status()
    except httpx.HTTPError as exc:
        return None, f"Failed to reach exchange rate service: {exc}"

    data = response.json()

    if data.get("result") == "error":
        return None, f"API error: {data.get('error-type', 'unknown-error')}"

    rates = data.get("conversion_rates")
    if not rates:
        return None, "Unexpected response: no conversion rates returned."

    return rates, None

Step 5. Add the three currency tools

Python
@mcp.tool()
def get_exchange_rate(currency: str) -> str:
    """Get the live exchange rate from the given currency to EUR using ExchangeRate-API."""
    base_currency = currency.upper()

    rates, error = _fetch_conversion_rates(base_currency)
    if error is not None:
        return error

    rate = rates.get("EUR")
    if rate is None:
        return f"No EUR exchange rate available for {base_currency}."

    return f"1 {base_currency} = {rate} EUR"


@mcp.tool()
def get_live_exchange_rate(from_currency: str, to_currency: str = "EUR") -> str:
    """Get the live exchange rate between two currencies using ExchangeRate-API.

    Args:
        from_currency: ISO 4217 base currency code, for example USD or GBP.
        to_currency: ISO 4217 target currency code, defaults to EUR.
    """
    from_currency = from_currency.upper()
    to_currency = to_currency.upper()

    rates, error = _fetch_conversion_rates(from_currency)
    if error is not None:
        return error

    rate = rates.get(to_currency)
    if rate is None:
        return f"Unsupported target currency: {to_currency}"

    return f"1 {from_currency} = {rate} {to_currency}"


@mcp.tool()
def convert_currency(amount: float, from_currency: str, to_currency: str) -> str:
    """Convert an amount from one currency to another using live exchange rates.

    Args:
        amount: The amount of money to convert.
        from_currency: ISO 4217 currency code to convert from, for example EUR or USD.
        to_currency: ISO 4217 currency code to convert to, for example USD or GBP.
    """
    from_currency = from_currency.upper()
    to_currency = to_currency.upper()

    rates, error = _fetch_conversion_rates(from_currency)
    if error is not None:
        return error

    rate = rates.get(to_currency)
    if rate is None:
        return f"Unsupported target currency: {to_currency}"

    converted_amount = amount * rate
    return (
        f"{amount:.2f} {from_currency} is approximately "
        f"{converted_amount:.2f} {to_currency}. "
        f"Exchange rate: 1 {from_currency} = {rate} {to_currency}"
    )

Bonus Exercise: Add a Resource to the MCP server

MCP supports resources as a first-class concept, separate from tools. A resource is read-only content (for example Markdown documentation) that the AI client can fetch by URI. The final main.py exposes its documentation this way instead of as a tool.

Steps

  1. Make sure resources/exchange-rate-resource.md exists (it ships with this repo).
  2. Add from pathlib import Path to the imports in main.py.
  3. Register the file as an MCP resource with the @mcp.resource decorator:
Python
from pathlib import Path


@mcp.resource(
    "resources://exchange-rate-resource",
    name="Exchange Rate Resource",
    description="Markdown documentation with examples, currency codes, use cases, and common errors for the exchange rate MCP server."
)
def get_exchange_rate_resources() -> str:
    """Return the Markdown resource file for the exchange rate MCP server."""
    return Path("resources/exchange-rate-resource.md").read_text(encoding="utf-8")
The resources:// URI is how MCP clients reference a resource. The repo also registers a second resource, resources://profile, the same way using resources/profile.md.

Example usage in Copilot Chat:

Prompt
Read the resources://exchange-rate-resource resource and summarize it.

Using @mcp.resource instead of a tool keeps read-only content out of the tool list and lets clients reference it by URI. For production, add a permissions layer.

Bonus Exercise: Make the live API integration robust

The currency tools you added in the challenge all share one helper, _fetch_conversion_rates. Centralizing the network call in a single helper is what makes the integration reliable. Review how it handles the three things that go wrong with live APIs:

Missing API key Returns a clear message instead of crashing when EXCHANGE_RATE_API_KEY is not set.
Network or HTTP errors httpx.HTTPError is caught and turned into a readable tool response.
API-level errors Checks result == "error" and a missing conversion_rates field.
Why it matters The AI receives a helpful message, not a stack trace, so it can explain the problem to the user.

Test the error path by temporarily removing the key from .env and restarting the server:

Prompt
What is the USD to EUR exchange rate?
Expected
API key is missing. Set EXCHANGE_RATE_API_KEY as an environment variable.

Tip: when integrating live APIs, handle network errors, timeouts and unexpected responses. For production, add caching, rate-limit handling, and retries with backoff.
