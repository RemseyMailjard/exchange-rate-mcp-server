- Build and maintain backend services for digital banking features.
- Design APIs used by web, mobile, and internal Rabobank applications.
- Integrate backend systems with payment, customer, and account services.
- Implement secure authentication and authorization flows.
- Improve performance, resilience, and monitoring of production services.
- Participate in code reviews, refinement sessions, and architecture discussions.
- Collaborate with frontend developers, product owners, testers, security specialists, and platform engineers.

## Example Current Project

Mark is working on a customer profile service that gives authenticated users access to their personal banking information across Rabobank channels. The service exposes REST APIs, validates permissions, reads customer data from internal systems, and publishes events when customer details are updated.

## Working Style

Mark prefers simple, well-tested solutions over unnecessary complexity. He documents API contracts clearly, keeps services small and focused, and pays close attention to security and auditability. Because he works in a banking environment, he treats data privacy, operational stability, and compliance as everyday engineering concerns.

## Goals

- Modernize legacy backend services into cloud-native .NET APIs.
- Reduce deployment risk through better automated testing and observability.
- Improve developer experience by standardizing API templates and shared libraries.
- Help junior developers grow through mentoring and practical code reviews.
